# 🌌 VibeFetch

**VibeFetch** — это минималистичная и быстрая утилита для вывода информации о системе, написанная на **.NET 8**. Она создана для тех, кто ценит эстетику **Catppuccin** и стабильность в работе.

\!

## ✨ Особенности

  * 🐧 **Кроссплатформенность**: Стабильная работа на Windows, Fedora, CachyOS и других Arch-based дистрибутивах.
  * 🎨 **Catppuccin Theme**: Приятная цветовая палитра для твоего терминала.
  * 🛠 **Single-File**: Весь инструмент собран в один исполняемый файл без лишних зависимостей.
  * 🚀 **Smart Install**: Автоматическое добавление в системный PATH для Windows.

## 📊 Выводимые данные

  * **OS**: Название дистрибутива или версии Windows.
  * **Kernel**: Версия ядра системы.
  * **Uptime**: Время работы системы с момента запуска.
  * **RAM**: Используемая и общая оперативная память.
  * **Disk**: Заполненность системного диска.

## 🚀 Установка

### Windows

1.  Скачайте `vfetch.exe` из [Releases](https://www.google.com/search?q=../../releases).
2.  Запустите терминал от имени администратора в папке с файлом.
3.  Выполните команду для установки в систему:
    ```bash
    ./vfetch.exe install
    ```
4.  Перезапустите терминал и просто введите `vfetch`.

### Linux (Fedora / CachyOS / Arch)

1.  Скачайте бинарный файл `vfetch` из [Releases](https://www.google.com/search?q=../../releases).
2.  Дайте файлу права на выполнение:
    ```bash
    chmod +x vfetch
    ```
3.  Переместите его в системную папку:
    ```bash
    sudo cp vfetch /usr/local/bin/
    ```
4.  Теперь утилита доступна из любой папки по команде `vfetch`.

## 🛠 Сборка из исходников

Если вы хотите собрать проект самостоятельно, убедитесь, что у вас установлен **.NET 8 SDK** и **Python 3**.

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/Minish777/VibeFetch.git
    cd VibeFetch
    ```
2.  Запустите скрипт автоматической сборки:
    ```bash
    python builder.py
    ```
3.  Готовый бинарник появится в папке `publish/`.

-----

## 🔧 Техническая информация

Для обеспечения стабильности на Linux (особенно на Fedora) проект использует специфические флаги публикации:

  * `IncludeNativeLibrariesForSelfExtract`: позволяет избежать ошибок переполнения при чтении ресурсов.
  * `EnableCompressionInSingleFile=false`: гарантирует совместимость с различными менеджерами памяти в Linux.
